# SOFTWARE ARCHITECTURE

La aplicación debe construirse como un sistema real y escalable.

Requerimientos:
- Frontend desacoplado del backend
- REST API
- Arquitectura modular
- Clean Architecture
- Separación de responsabilidades
- DTOs
- Services
- Repository pattern
- Middleware
- Authentication layer
- Validation layer
- Error handling
- Logging
- Database migrations
- Environment variables
- Docker support

Debe ser mantenible y preparada para crecimiento futuro.
